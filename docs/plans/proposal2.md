# 소개퐝 기획서

## 개요

포항에 위치한 대학교 학생들끼리 소개팅을 위한 서비스이다.
포항에 위치한 대학교는 포스텍, 한동대학교, 선린대학교, 포항대학교이다.

어드민이 특정 기간동안만 소개팅 참여 신청을 받는다.
신청시 학교 이메일로 가입하여 포항에 소속된 대학교임을 인증한다.
필요한 인원수가 채워지면 소개팅 신청이 종료된다.
신청한 유저들은 자신의 정보와 자신이 원하는 상대의 정보를 기입한다.
기입한 정보에 가중치를 부여해 가중치 알고리즘으로 '선호도 순위' 생성을 1단계로 한다.
게일-섀플리 알고리즘으로 '매칭 확정'
매치가 확정되면 이메일로 알림을 보낸다. 신청할때 작성한 연락 수단(전화번호, 카카오톡, 인스타그램 등)을 같이 보내서 상대방에게 연락할 수 있도록 한다.

## 서버

서버는 아직 정해지지 않음
상황에 따라 수퍼베이스나 직접 로컬 데이터베이스 운영 고려

## 상세 기능 명세서

### 대시보드

- 왼쪽 상단에 소개퐝 서비스명 기입 그리고 클릭시 대시보드로 이동
- 오른쪽 상단에 로그인 버튼
- 메인 바디에는 서비스 소개하는 한줄과 오른쪽에 사진
- 그 아래에는 소개팅 참여기간과 현채 참여 인원 표시 (예시: 남자 2/10 여자 4/10)
- 소개팅 참여하기 버튼
- 현재 참여 가능한 대학교 로고들 가로로 나열

### 로그인 & 회원가입

- 로그인은 학생 이메일로만 가능하도록 할것
- 구글로그인만 제공

### 회원가입후

- 처음 회원가입했다면 본인 기본 정보와 원하는 이상형 정보 기입 폼 작성
- 본인정보란
  - 닉네임 (서비스에서 사용할 닉네임)
  - 나이 (년도 선택)
  - 성별
  - 학교 (이메일 도메인에 따라 자동 지정)
  - 학과/학부
- 기존에 작성했던 본인 외형정보, 라이프스타일, 상대 이상형 등은 여기서 작성하지 않음
- 회원가입 후에는 기본 정보만 작성하도록 한다

### 소개팅 생성

- 기존 명세서와 다르게 해당 서비스를 개발 할 계획
- 어드민은 어드민 페이지에서 소개팅을 생성하는데 다른 종류의 소개팅을 생성 할 수 있다. 소개팅 종류에 따라서 상세 기능들이 달라진다.
  - 블라인드 소개팅 (온라인)
    - 기존에 이 서비스가 할려고 했던 소개팅 여기서 더 추가 기능이 생김
  - 블라인드 소개팅 (오프라인)
  - 로테이션 소개팅
  - 그 외 등등

### 블라인드 소개팅 (온라인) -> MVP

- 블라인드 소개팅을 신청하면 기존에 있는 것 처럼 본인의 프로필을 완성하고 원하는 이상형을 작성한다. 여기서 작성한 블라인드 프로필은 저장되어서 다음에도 사용가능.
- 소개팅을 신청할때 본인 사진을 첨부 할 수 있도록 한다. 이 사진은 필수가 아니고 지원자가 원하면 제출한다. 안내사항으로 필수가 아니란걸 명시한다. 또한 블라인드 소개팅의 사진 제출은 주선자만 필요에 따라 확인 가능하며 그 의도와 목적은 신청자들이 작성한 본인의 프로필과 실제로 어느정도 일치한지 확인하기 위함이다. 기본적으로 매칭은 알고리즘으로 자동으로 되지만 사진을 올리게 되면 어느정도 주선자가 확인하여 작성한 프로필과 맞는지 확인하는 절차를 걸쳐 최종 매칭에 변경 될 수도 있다. 이부분을 기분 나쁘지 않게 명시해야함.
- 매칭이 종료되면 이메일로 참가자들에게 알림이 간다.
- 매칭된 커플수 만큼 채팅방을 만들어야 한다. 이 채팅방은 주선자(어드민)과 매칭된 커플이 함께 있는 톡방이다.
- 어드민은 각 톡방을 한눈에 볼 수 있는 전용 대시보드가 필요하다. 이 대시보드에서 어드민인 모든 톡방에 동시에 채팅을 보낼 수 있다. 그리고 새 채팅이 올라오면 UI로 표시된다. 특정 톡방을 선택해서 그 톡방에서만 또 따로 채팅을 할 수 있다.
- 충분한 시간이 지나면 어드민은 각 커플들에게 개인 연락처를 넘기고 소개팅을 더 진행할 의지가 있는지 묻는다.

### 소개팅 신청 페이지

- 현재 진행 중인 소개팅 정보 보여줌
- 기간과 모집 인원 그리고 현재 인원 표시
- 신청 버튼
- 안내사항
- 신청버튼을 누르고 기존 기본 프로필 정보가 맞는지 확인

### 프로필 페이지

- 자신의 프로필과 자신이 기입한 원하는 이상형 정보 보기 가능
- 자신이 신청한 소개팅 정보 표시, 매칭된 상대 정보 확인 가능
- 회원탈퇴 가능

### 어드민 페이지

- 회원 관리
- 소개팅 열기/닫기
- 어드민 기능들
- 소개팅 종류별로 소개팅을 오픈 할 수 있어야 한다.
- 소개팅을 오픈할때 제목, 신청 기간, 인원, 안내사항 등을 기입해야 한다.
- 소개팅을 오픈할때 모든 이메일 도메인이 지원 가능하게 하거나 성별마다 특정 도메인만 신청가능하게 하도록 한다
- 신청시 사진 첨부가 가능하게 할건지 필수인지 선택인지 정한다.

### 매칭 시스템 (서버와 연계)

- 소개팅 참여 기간이 지나기 전에 참여 인원이 다 차면 자동 신청 종료
- 참여 기간이 지났는데도 인원이 부족하면 어드민이 기간을 연장하거나 인원을 줄임
- 신청이 종료되면 자동으로 가중치 알고리즘으로 '선호도 순위' 생성
- 게일-섀플리 알고리즘으로 '매칭 확정'
- 유저에게 이메일로 알림 보내고 프로필 페이지에서도 확인 가능
